# Development Plan

For project mission, goals, and non-goals, see [GOALS.md](./GOALS.md).

## Work tracking

This project **tracks its roadmap as issues** in the GitHub tracker, not in this
file. Managed by `/do:replan --issues`; worked with `/do:next --issues` (or just
`/do:next` — an empty backlog here auto-redirects to issue mode).

- **Browse the backlog:** [open issues](https://github.com/atomantic/slashdo/issues)
  (the `plan`-labelled ones are the curated queue `/do:replan --issues` maintains).
- **Add work:** open an issue, or let `/do:replan --issues` / `/do:better` file one.
- **Claim the next item:** `/do:next --issues` (or `--swarm` to drain several in parallel).

This section intentionally contains no `- [ ]` checkboxes — the tracker is the
single source of truth, so PLAN.md doesn't churn while work happens on issues.
