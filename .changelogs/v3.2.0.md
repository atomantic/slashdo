# Release v3.2.0

Released: 2026-05-24

## Added

- `lib/finding-disposition.md`: shared guidance for PR-fixing review flows. Defines three dispositions for any review finding (fix-now / reply / defer), makes **fix-in-this-PR the default**, and restricts PLAN.md deferral to findings that are genuinely large/architectural or too risky to land in the current change — explicitly guarding against using PLAN.md as a dumping ground for fixable work. Loaded by `/do:rpr` and `/do:review`.
- **Capability-gated command rendering.** The transformer now resolves `<!-- if:teams -->…<!-- else -->…<!-- /if:teams -->` conditional blocks per environment against the `supportsTeams` capability flag in `src/environments.js` (the flag was previously defined and tested but had no consumer). The matching branch is kept and the markers stripped for each target environment. Documented in the README "Contributing" section.

## Changed

- `/do:rpr`: added a Notes bullet directing the agent to fix findings in the current PR and defer to PLAN.md only when a fix is genuinely large/risky.
- `/do:review`: load the finding-disposition guidance in the local-mode "Fix Issues" phase and tie the IMPROVEMENT-fix step to it.
- `/do:better` and `/do:better-swift`: parallel remediation (and its teardown) now uses `TeamCreate`-based teammates on Claude Code and falls back to the parallel sub-agent pattern (as used by `/do:rpr`) on OpenCode/Gemini/Codex — so both commands work across all four environments instead of only team-capable ones. The shared `lib/remediation-agent-template.md` is now mode-neutral.
- `lib/code-review-checklist.md`: added generation-counter guards for optimistic async resolution, and a check flagging divergent copies of a changed command-invocation string.
