# Release v3.13.0

Released: 2026-06-17

## Added

- **`--review-mode <series|parallel>` for the multi-reviewer review loop (default `series`).** When `--review-with` lists more than one reviewer, they now run **in series by default** â€” each reviewer runs to completion (review â†’ fix â†’ verify â†’ push) before the next starts, so a later reviewer reviews against the earlier ones' committed fixes and can catch problems a fix introduced. This makes the reviewer list's order meaningful (put the highest-signal reviewer first) and is why series is the default. The opt-in `parallel` mode runs every reviewer's review concurrently against one frozen baseline and then applies the deduped union of findings in a single pass â€” faster, but no reviewer sees another's fixes (and `--reviewer-applies` / the stop-modes are ignored, since concurrent reviewers can't share a working tree and there's no first-finisher to stop on). Available on `/do:pr`, `/do:review`, `/do:better`, `/do:better-swift`, `/do:depfree`, and `/do:release` (and forwarded by `/do:next` and `/do:pr-better`); settable as a saved default via `/do:config --review-mode <series|parallel>`. `/do:rpr` is unaffected (its parallelism is review-thread resolution, not reviewer dispatch).

## Fixed

- The `agy` local-agent reviewer (`--review-with agy`, also reached via `gemini`/`antigravity`) no longer hangs and returns zero findings. The review loop used to invoke slashdo's `/do-review` skill, which fans out to 5â€“6 parallel sub-agents â€” but `agy -p` (print mode) can't wait on that fan-out: it returns the orchestrator's interim "I dispatched the sub-agents" message and then times out (`Print mode: timed out after 498 polls`) without ever printing their aggregated findings. The local-agent loop now drives `claude`/`agy` with a self-contained single-agent review prompt (carrying its own `git diff` instruction and output contract, and explicitly forbidding sub-agent dispatch) instead of the skill, and adds `--print-timeout 30m` to the agy invocation so a real multi-file review isn't cut off on stock macOS (which has no `timeout`/`gtimeout`). The same prompt also makes the in-process Claude sub-agent path safe from a nested skill fan-out. `codex` (built-in `codex review`) and `ollama` were already skill-free and are unchanged.
- The GitHub auth pre-flight in `/do:next`, `/do:pr`, `/do:replan`, `/do:better`, `/do:better-swift`, `/do:depfree`, and the shared issue-mode setup now checks `gh auth status --active` instead of a bare `gh auth status`. A bare check exits non-zero whenever *any* configured `gh` account has a stale or invalid token â€” even when the account you're actually using is authenticated fine â€” which made these commands falsely report "not authenticated" and run extra diagnostic steps on every invocation. Scoping the check to the active account fixes that.
