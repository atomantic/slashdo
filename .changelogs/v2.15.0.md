# Release v2.15.0

Released: 2026-05-17

## Added

- **Plan-item ID system for parallel-safe agent work.** Every `- [ ]` checkbox in PLAN.md now carries a stable `[<slug>]` ID so concurrent agents can claim distinct items via their worktree branch name (`cos/<task>/<slug>/<agent>`) and detect what's in flight by scanning git branches and open PRs for the slug. New shared spec at [`lib/plan-id-format.md`](../lib/plan-id-format.md) defines slug derivation (markdown-strip → kebab-case → 50-char cap on word boundary → collision-resolve against PLAN.md + DONE.md). `do:replan` gains a **Phase 0** that assigns IDs to every checkbox missing one before any other work runs; Phase 5 preserves existing IDs verbatim (slugs are immutable once assigned). Phase 4 archive entries to DONE.md now carry the slug as `- **[<slug>] Title** — description` so retired IDs are never recycled.
- `do:replan` now performs drift detection: a new agent checks each still-pending PLAN.md item against recent commits to flag cases where executing the item as written would remove or regress a feature added since the item appeared. Drifted items are never auto-modified — autonomous mode annotates them with a `> ⚠️ DRIFT:` blockquote (collision + commit SHA) and surfaces them in the summary; `--interactive` mode walks each one individually with three explicit human choices: **replan** (drafts a rewrite for approval), **examine** (leaves annotated), or **delete**.

## Changed

- `do:push` and `do:pr-better` now preserve the `[<slug>]` ID when flipping `- [ ]` → `- [x]` on completed items — only the box character changes, the slug stays. Commit messages and changelog entries SHOULD reference the slug (e.g. `feat([<slug>]): …`) so finished work is grep-able across DONE.md, branches, and PR titles.
- `do:better`, `do:better-swift`, and `do:depfree` now require every appended audit `- [ ]` line to carry a unique `[<slug>]` ID. Recommended slug patterns documented per command (`[sec-<file>-<hint>]`, `[drop-<package>]`, etc.) so concurrent fix-up agents can split categories without collision.
- `do:goals` migrates checkbox task lists from GOALS.md into PLAN.md with a slug assigned at insert time.
- `do:rpr` now uses a single persistent `Monitor` for the entire session — one stream that emits an event for each new Copilot review AND each CI bucket transition. Previously the skill implicitly encouraged spawning a fresh background poll per review loop iteration, which produced 5+ active subshell tasks across a multi-round PR. The "CI Health Check During Review Polling" section is collapsed into a tighter "CI failure handling" section since the monitor surfaces failures as events without a separate poll.
