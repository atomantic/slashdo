# Release v3.22.0

Released: 2026-07-09

## Highlights
- **Grok Build support.** slashdo now installs into xAI's Grok Build CLI (`grok`) — detected automatically when `~/.grok` exists, or target it explicitly with `npx slash-do@latest --env grok`. Commands install as Agent Skills under `~/.grok/skills/`, the same SKILL.md format used for Codex.
- **Review loops now converge instead of mining edge cases.** A new per-round convergence gate lets a reviewer judge, each round, whether another review pass is actually worth it — so a capable reviewer stops after the substantive findings are fixed instead of spiraling toward the iteration cap inventing ever-more-marginal edge cases.
- **Curl uninstaller reaches parity with the npm installer** for directory-namespaced Agent Skills environments — it now cleans up Codex and Grok Build installs, not just Claude Code / OpenCode / Antigravity.

## Added
- **`grok` environment** — installs commands as Agent Skills under `~/.grok/skills/<do-cmd>/SKILL.md`, the same SKILL.md-per-directory format used for Codex and the Antigravity CLI (Grok Build auto-loads skills from that directory). Like those environments, install requires Node.js (`npx slash-do@latest --env grok`) because lib content is inlined into each skill; the curl installer detects Grok Build and points users at the npx command. The config-path token is rewritten to `~/.grok/.slashdo-config.json` so `/do:config` defaults resolve correctly.
- **Per-round review convergence gate** (`lib/review-convergence-gate.md`) — a judgment gate (not a new hard limit) run at the top of each re-review decision in all four inner review loops (local-agent, ollama, copilot, github-reviewer). If the round just completed made zero commits or landed only marginal findings, the loop converges and stops; only a round with a substantive finding earns another pass, and two consecutive marginal-only rounds is a definitive converge signal regardless of round number. The mechanical `MAX_ITERATIONS` cap stays as a runaway backstop the gate should fire before. The gate governs a single reviewer's self-re-review only — it never overrides the user's cross-reviewer `--review-stop-*` choice or skips a requested reviewer.

## Fixed
- **Curl uninstaller now cleans up directory-namespaced Agent Skills environments.** `uninstall.sh` gained a shared helper that removes `~/.<env>/skills/do-<cmd>/` skill directories plus the env's `.slashdo-version` / `.slashdo-config.json`, wired up for both **Grok Build** (`~/.grok`) and **Codex** (`~/.codex`). Previously the curl uninstall path detected and cleaned only Claude Code, OpenCode, and Antigravity — a Codex user (and now a Grok Build user) following the documented `uninstall.sh` route was left with orphaned skill files. The npm uninstaller (`npx slash-do@latest --uninstall`) already handled these envs; this brings the curl path to parity.
- **Ollama convergence honors the coverage-gap invariant.** The new convergence gate set `STATUS=clean` on converge, but the ollama loop's coverage-gap invariant (a partially reviewed diff is never `clean`) still holds at the gate: a round can converge with applied findings *and* errored/truncated files. The gate now reports `incomplete` when `REVIEW_ERRORS + TRUNCATED > 0`, keeping a partially-reviewed diff out of `--merge` eligibility.
