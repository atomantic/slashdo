# Release v2.17.0

Released: 2026-05-19

## Added

- **`--review-with copilot|codex|gemini|claude` flag on `do:pr` and `do:release`** (and pass-through in `do:pr-better`). Selects which agent performs the post-PR code review. `copilot` (default) preserves the existing Copilot cloud review loop; `codex`, `gemini`, and `claude` route to a new local-agent review loop in [`lib/local-agent-review-loop.md`](../lib/local-agent-review-loop.md). The local loop invokes the chosen CLI in **headless reckless mode** ‚Äî `claude --dangerously-skip-permissions`, the built-in `codex review` subcommand (already non-interactive by design), or `GEMINI_SANDBOX=false gemini --yolo` ‚Äî captures output to a tempfile log, and runs unattended. For `claude` and `gemini`, the invocation calls slashdo's installed `/do:review` command directly with a one-line override appended to switch commit messages to `address review: <summary>` and skip pushing. For `codex`, the loop uses codex's first-class **`codex review` subcommand** (rather than re-prompting through `codex exec`), so the review is authentically codex-flavored; if `codex review` exits without applying fixes, the orchestrating agent reads the review log and applies fixes itself before continuing. The orchestrating (main) agent then **verifies in-thread before pushing** ‚Äî it inspects every new commit's diff for out-of-scope refactors, disabled tests, and secrets, runs the project build + tests, and only on a clean pass runs `git push`; any verification failure triggers `git reset --hard` back to the loop's baseline SHA and reports a non-clean status. Up to 3 iterations to catch recursive findings, then `guardrail`-stops. `do:release`'s merge gate now branches per agent: copilot keeps the "zero comments" rule, local-agent only merges on `STATUS=clean` (`guardrail` / `cli-error` / `broken-build` / `test-failed` / `rejected` block the merge in default mode).

## Changed

- **Codex invocation in the local-agent review loop now uses the base-branch review target**: `codex review --base "$BASE_BRANCH" --title "$REVIEW_TITLE"`. Previously the lib doc only said `codex review --base <BRANCH>` and noted that flag names "may vary by codex version" ‚Äî which sent every agent that ran the loop on a doc-spelunking detour. Codex review's three review targets (`--uncommitted`, `--commit <SHA>`, `--base <BRANCH>`) are mutually exclusive, and `--base` is *also* mutually exclusive with the positional `[PROMPT]`; combining them makes codex exit with code 2 before any review runs. The lib now precomputes just `$REVIEW_TITLE` (cosmetic ‚Äî codex displays it in the review summary), explains the mutual-exclusion rules in-line, and notes that the commit-style override that `claude`/`gemini` receive via prompt cannot be passed to codex ‚Äî the orchestrating agent supplies the `address review: <summary>` commit message itself when applying findings from codex's review log. The invocation also omits `-a never` ‚Äî `codex review` is non-interactive by design (per its `--help`) and rejects the flag.

## Fixed

- **`install.sh` and `uninstall.sh` LIBS arrays** include `local-agent-review-loop` so the curl-bash install path (and local-mode `./install.sh`) actually delivers the new file to `~/.claude/lib/` and `~/.gemini/lib/`. Without this entry, `do:pr --review-with codex|gemini|claude` failed at the `!cat ~/.claude/lib/local-agent-review-loop.md` inclusion step with `No such file or directory`. Note: the `npx slash-do` installer is unaffected ‚Äî it auto-discovers files in `lib/` via `collectLibFiles()`. Only the bash installer carries a hardcoded list.
- **Gemini invocation now works under the `timeout 1800` wrapper.** The table cell was `GEMINI_SANDBOX=false gemini --yolo ‚Ä¶`, which becomes `timeout 1800 GEMINI_SANDBOX=false gemini ‚Ä¶` when the wrapper is applied. Shell `VAR=value cmd` env-assignment syntax only works at the head of a command line ‚Äî after `timeout`, bash treats `GEMINI_SANDBOX=false` as the executable name and exec fails before gemini ever runs, returning `cli-error` instead of running the review. The cell now uses `env GEMINI_SANDBOX=false gemini --yolo ‚Ä¶`, which delegates the env assignment to `env(1)` and works correctly under the wrapper.
- **`REVIEW_AGENT` is reassigned to `copilot` when falling back.** Pre-flight step 2 now mandates resetting `REVIEW_AGENT=copilot` before invoking the Copilot loop. Previously, if the requested local CLI (codex/gemini/claude) was missing, the fallback ran the Copilot loop but left `REVIEW_AGENT` pointed at the missing agent. The downstream merge gate in `do:release` dispatches on `REVIEW_AGENT` and only accepts local-agent `STATUS=clean`, so a clean Copilot fallback review would still block the release from merging in default mode.
- **Claude/Gemini slash-command argument no longer carries override prose.** The original cells passed `/do:review {BASE_BRANCH} ‚Äî commit each fix batch ‚Ä¶` as a single argument to `do:review`, which reads `$ARGUMENTS` as the base branch and runs `git diff $ARGUMENTS...HEAD`. The diff then resolved against a non-existent ref like `"main ‚Äî commit each fix batch ‚Ä¶"` and failed before any review work. The lib now precomputes `LOCAL_PROMPT` via `printf '/do:review %s\n\n%s' "$BASE_BRANCH" "$REVIEW_OVERRIDE"`, so the slash command receives only the branch and the commit-style override travels as a separate sentence on a new line where the agent can read it as instructions.
- **Timeout wrapper is selected at pre-flight, not assumed.** Stock macOS does not ship `timeout(1)`; running the prescribed `timeout 1800 ‚Ä¶` line under that environment exited 127 before the reviewer started, returning `STATUS=cli-error` on every `--review-with` invocation. Pre-flight now probes for `timeout`, then `gtimeout`, then falls back to an empty `$TIMEOUT_CMD` that lets the CLI use its own internal limits. The step-2 invocation uses `$TIMEOUT_CMD {INVOCATION}` so an empty wrapper expands cleanly to a direct call.
