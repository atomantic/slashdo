# Release v3.12.0

Released: 2026-06-15

## Added

- `/do:config --issues` saves an **issue-mode default** so every command that accepts `--issues` (`/do:next`, `/do:replan`, `/do:better`, `/do:better-swift`, `/do:depfree`, `/do:review`, `/do:rpr`) defaults to filing/working tracker issues instead of `PLAN.md`. `--no-issues` opts out for a single run; `--issues-label <name>` saves the scoping label. Resolved with the same global + per-project precedence as the review-loop defaults.
- `/do:pr --merge` — opt-in auto-merge: once the review loop returns a mergeable status **and** required CI checks pass, the PR is merged (GitHub-native auto-merge, falling back to an in-session check-watch; GitLab via merge-when-pipeline-succeeds). `--merge=<method>` / `--merge-method <squash|rebase|merge>` pin the method (default: the repo's allowed method), `--no-merge` opts out, and `/do:config --merge` / `--merge-method` save it as a default. Scoped to `/do:pr`; the other merge-capable commands keep their own behavior.

## Changed

- `/do:next` now passes `--no-merge` to its `/do:pr` sub-invocation so a global `merge` default can't merge a PR out from under `/do:next`'s own merge gate and worktree/issue cleanup.
- Issue-mode commands now gate on the resolved `ISSUE_MODE` (flag **or** saved default) rather than the literal `--issues` flag, so a saved `issues=true` default takes effect end-to-end.
- README documents the new saved defaults and adds a dedicated `/do:pr` auto-merge section.
