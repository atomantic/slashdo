# Release v2.14.0

Released: 2026-05-13

## Added

- **Post-review documentation recommendations.** After `do:review`, `do:rpr`, and the end of a Copilot review loop, the agent now scans the findings for patterns worth surfacing in project documentation (CLAUDE.md / AGENTS.md, CONTRIBUTING.md, README.md, in-tree comments). New `lib/post-review-doc-recommendations.md` defines what counts as a doc candidate (preventable by knowing a convention; non-obvious from code; likely to recur) versus what doesn't (one-off bugs, fixes that are self-documenting, generic best practices). All recommendations are paste-ready suggestions only — never auto-edits.

## Changed

- **`do:rpr`: stop review loop when findings are only nitpicks.** Step 8 now evaluates whether another Copilot review round is worth requesting before looping. If all findings from the last round were trivial (style preferences, naming suggestions, "consider..." language, or repeats of already-dismissed feedback) with no correctness/security/logic issues and fewer than 3 actual code changes made, the loop exits with a message and proceeds to merge. Substantive findings (logic bugs, security issues, missing guards, contract violations) still trigger another round.
- **`do:fpr`: don't block PR on pure style nitpicks.** The local code review gate now includes a worthiness check — findings that touch correctness, security, or logic require fixes before opening the PR, but if ALL findings are style/naming nitpicks, proceed without blocking and note them briefly in the PR description.
- **`do:review`: triage Surface Scan agent into Always / Conditional groups.** Surface Scan was at ~150 items and overflowing the agent's attention budget. The agent now scans ~60 Always items on every file (Hygiene, Imports, Runtime correctness, Async & state, Error handling) and consults ~85 Conditional items only when a file in the diff matches a bracketed file-type trigger (`_[HTTP route files]_`, `_[SQL, ORM, migration files]_`, `_[SSE, WebSocket, chunked HTTP server handlers]_`, etc.). API route basics and Streaming response handlers moved from Always to Conditional.
- **`improve:review`: aligned with the actual 5-agent review system.** The command's Architecture Overview, agent-assignment guide, and Phase 3a / 4b / 4e / report template now reference all five agents (Surface Scan, Surface Quality, Security Audit, Cross-File Tracing, Cross-File Contract). Phase 2c now distinguishes Cross-File Tracing (data / control flow, state, lifecycle, concurrency) from Cross-File Contract (schema / shape / agreement).
- **Review checklist coverage broadened.** Master checklist and the five agent files learned new and broadened items for: empty-array / empty-string fallbacks that block users from clearing fields; cached TTL / expiry honored only at init; user-controlled command interpolation and argv-leak risk; `safe*`/`try*` named helpers that don't actually wrap their core; strict schemas rejecting shapes the runtime migrates; client / server action-gating drift; CRUD-verb parity within a single router; schema-tightening audits against existing callers; fire-and-forget startup work racing with first traffic; monkey-patch overrides covering some entrypoints but not their siblings; unknown-HTTP-failure classifier fall-through; derived-after-mutation booleans; UI scaffold-presence gating; normalize-both-sides-of-join; stale TODO / PLAN entries.
- **Review checklists generalized for cross-codebase use.** Removed project-specific identifiers (override / job / service names, trading-domain field examples, press / audio interaction examples, project-specific config filenames, project-specific test names) from the master checklist and agent files, replacing them with neutral generic equivalents so the checklists work on any project without leaking origin context.

## Fixed

- **`install.sh` / `uninstall.sh`: ship `post-review-doc-recommendations.md`.** Added the new lib file to the hardcoded `LIBS` arrays in both shell scripts so the curl-bash install path delivers it to `~/.claude/lib/`. Without this, `do:review`, `do:rpr`, and the Copilot review loop would `cat` a missing file at runtime on installs done via the shell bootstrap. The npm-installed `slash-do` CLI was unaffected — it auto-discovers `lib/*.md`.
