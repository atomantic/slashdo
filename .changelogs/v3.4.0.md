# Release v3.4.0

Released: 2026-05-30

## What's New

### Antigravity CLI replaces Gemini CLI (`agy`)

The Gemini CLI has been superseded by the Antigravity CLI (binary `agy`), which uses the Agent Skills format (`~/.gemini/antigravity-cli/skills/`) instead of TOML-header commands under `~/.gemini/commands/`. This is not a style change — the installed path, format, and binary all differ.

- Rename the `gemini` environment key to `antigravity` in environments.js; add `gemini` and `agy` as ALIASES so existing `--env` flags and `--review-with` slugs keep working without a breaking change
- Drop `toTomlHeader` and the `toml` format branch from transformer.js — Antigravity uses yaml-frontmatter (Agent Skills), so TOML is no longer needed by any environment
- Updated `install.sh`: detects `~/.gemini/antigravity-cli`, replaces `install_gemini` (TOML conversion + lib copy) with the Agent Skills note directing users to `npx` for content inlining
- Updated `uninstall.sh`: `uninstall_antigravity` removes skill directories under `~/.gemini/antigravity-cli/skills/` instead of flat `.md` files
- Updated `local-agent-review-loop.md`: `agy` invocation uses `--dangerously-skip-permissions -p` (replaces `gemini --yolo + env GEMINI_SANDBOX=false`); slash command is `/do-review` (Agent Skills naming, not `/do:review` colon-namespacing)

### Install/Uninstall fixes

- `install.sh`: the "Done! Commands are available..." message is no longer printed when only a referral was shown (Antigravity/Codex can't install via curl); a clarifying note now directs those users to complete the `npx` step
- `uninstall.sh`: users who installed under the legacy Gemini CLI path (`~/.gemini/commands/do/`) now get proper cleanup instead of "Nothing to uninstall" after upgrading
