## Fixed

- The `agy` local-agent reviewer (`--review-with agy`, also reached via `gemini`/`antigravity`) no longer hangs and returns zero findings. The review loop used to invoke slashdo's `/do-review` skill, which fans out to 5–6 parallel sub-agents — but `agy -p` (print mode) can't wait on that fan-out: it returns the orchestrator's interim "I dispatched the sub-agents" message and then times out (`Print mode: timed out after 498 polls`) without ever printing their aggregated findings. The local-agent loop now drives `claude`/`agy` with a self-contained single-agent review prompt (carrying its own `git diff` instruction and output contract, and explicitly forbidding sub-agent dispatch) instead of the skill, and adds `--print-timeout 30m` to the agy invocation so a real multi-file review isn't cut off on stock macOS (which has no `timeout`/`gtimeout`). The same prompt also makes the in-process Claude sub-agent path safe from a nested skill fan-out. `codex` (built-in `codex review`) and `ollama` were already skill-free and are unchanged.
- The GitHub auth pre-flight in `/do:next`, `/do:pr`, `/do:replan`, `/do:better`, `/do:better-swift`, `/do:depfree`, and the shared issue-mode setup now checks `gh auth status --active` instead of a bare `gh auth status`. A bare check exits non-zero whenever *any* configured `gh` account has a stale or invalid token — even when the account you're actually using is authenticated fine — which made these commands falsely report "not authenticated" and run extra diagnostic steps on every invocation. Scoping the check to the active account fixes that.
