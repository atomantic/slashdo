# Release v3.13.1

Released: 2026-06-17

## Fixed

- **Local-agent reviews no longer hit a 10-minute timeout.** `--review-with codex|agy|claude` reviews now launch in the background and poll a `$DONE_FILE` sentinel for completion, instead of a single blocking foreground call that the host CLI's Bash tool kills at its ~10-minute foreground ceiling (600000 ms on Claude Code). On stock macOS — where no `timeout`/`gtimeout` binary exists — that host cap, not `timeout 1800` or agy's `--print-timeout 30m`, was the bound a real multi-file `agy` review kept exceeding, getting killed with zero findings.

## Changed

- **Pin a fast agy review model.** The `agy` invocation now passes `--model "$AGY_REVIEW_MODEL"` (default `Gemini 3.5 Flash (High)`, override via the `AGY_REVIEW_MODEL` env var). agy's default can be a heavy "Thinking" tier that spends many minutes in hidden reasoning, turning a routine review into a 20–30 minute wait that looks like a hang. A capable fast model returns in well under a minute on a small diff; bump `AGY_REVIEW_MODEL` to a heavier tier when you want more depth (the background launch + 30-minute bound cover it).
- **Documented the agy latency/visibility behavior.** A quiet review log during the background poll is not evidence the reviewer is stuck — visible output is model-dependent (lighter models narrate actions; heavy thinking tiers emit nothing until the final answer). Only a `$DONE_FILE` exit code or a 30-minute overrun signals completion or failure. Also noted that prompting agy to shell out to `agy` itself (e.g. `agy models`) can stall on the nested invocation.
