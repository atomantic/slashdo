# Release v3.1.2

Released: 2026-05-23

## Fixed
- **Codex local-agent review loop now runs with `--sandbox danger-full-access`.** Both codex invocations in `lib/local-agent-review-loop.md` — review-only (`codex review`) and reviewer-applies (`codex … exec`) — now pass the top-level `--sandbox danger-full-access` flag before the subcommand. Codex's default `workspace-write` sandbox blocks network and restricts command execution, so without the flag `codex review` couldn't reliably read the working tree or run git (producing no usable findings) and the apply path couldn't run build/tests/network ops. `--sandbox` is a top-level option (like `-a`) and must precede the `review`/`exec` subcommand; the two combine as `codex --sandbox danger-full-access -a never exec …`. Mirrors the unrestricted posture already used for `claude --dangerously-skip-permissions` / `gemini --yolo`. The `do:review` PR-mode skip note is updated to the new invocation strings.
