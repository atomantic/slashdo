# Unreleased Changes

## Added

- `--review-iterations <n>` flag to cap how many Copilot review-and-fix cycles run. Accepted by `do:pr`, `do:release`, `do:review`, `do:pr-better` (forwarded to `do:pr`), and the direct-loop audit commands `do:better`, `do:better-swift`, and `do:depfree`. `0` restores the legacy "loop until 0 comments" behavior (bounded by the 10-iteration safety guardrail); positive `n` runs at most `n` cycles, still exiting early on a zero-comment review.
- New Copilot loop terminal status `capped` — the configured iteration cap was reached after applying every fix the review surfaced. Treated as clean-equivalent for merge purposes (rolls up into the multi-reviewer wrapper's `OVERALL_STATUS=clean`).

## Changed

- **`--review-with` no longer defaults to `copilot` — there is no default reviewer.** On `do:pr`, `do:release`, and `do:pr-better`, omitting `--review-with` now runs **no** external review loop (each command still runs its own unconditional self-review gate); previously these defaulted to `REVIEW_AGENTS=[copilot]`. Whatever you list is exactly what runs, in order — `--review-with codex` runs codex only; `copilot` is never added implicitly. The multi-reviewer loop's missing-binary fallback that appended `copilot` when every requested local CLI was absent has also been removed: a missing reviewer is recorded `skipped` (→ `inconclusive` aggregate), never silently replaced by copilot. Affects `commands/do/pr.md`, `commands/do/release.md`, `commands/do/pr-better.md`, `lib/multi-reviewer-loop.md`, and `README.md`.
- **`do:better`, `do:better-swift`, and `do:depfree` now route their post-PR review step through the shared `--review-with` multi-reviewer loop instead of a hardcoded Copilot loop.** They accept `--review-with`, `--review-stop-on-findings`/`--review-stop-on-clean`, and `--reviewer-applies` (in addition to the existing `--review-iterations`). With **no** `--review-with`, the review loop **and** the auto-merge are skipped and PRs are left open for manual review — auto-merge now requires an explicit reviewer. `copilot` is never added implicitly; a missing local CLI is recorded `skipped`, not replaced by copilot.
- **`do:rpr` now resolves review threads from any author and no longer hardcodes Copilot.** Thread resolution is reviewer-agnostic (Copilot, human, or other bot). rpr requests/re-requests a **Copilot** review only when Copilot is already the reviewer in play (an existing/pending Copilot review) or when the PR has no review at all — a PR reviewed only by a human is resolved without summoning Copilot. `--review-with codex|gemini|claude` swaps in the local-agent review loop instead of a Copilot cloud review; `--reviewer-applies` is forwarded to that loop.
- **Default Copilot review behavior is now a single review-and-fix pass** (`--review-iterations` defaults to `1`) instead of looping until Copilot returns zero comments. Request one review, apply all its fixes, then stop. The unlimited-loop behavior remains available via `--review-iterations 0`. Affects `lib/copilot-review-loop.md`, `lib/multi-reviewer-loop.md`, and every command that runs the Copilot loop. The 10-iteration safety guardrail now applies only in unlimited mode.
- **The `claude` reviewer in the local-agent review loop now runs as an in-process sub-agent under Claude Code, instead of shelling out to `claude -p`.** A headless `claude -p` session bills against the Anthropic API even when the host session is already on a plan; dispatching a sub-agent via the `Agent` tool keeps the review on plan billing. Gated by the `if:teams` transformer conditional in `lib/local-agent-review-loop.md`, so only the Claude Code build uses the sub-agent path — Gemini/Codex/OpenCode builds keep the `claude -p` fallback (the Claude API billing there is unavoidable). The sub-agent receives the same `$LOCAL_PROMPT` and its returned message is captured to the loop's log file, so the review/verify/push mechanics are otherwise unchanged.
