# Release v2.18.0

Released: 2026-05-19

## Added

## Changed

- **`do:replan` no longer maintains a `DONE.md` archive.** Completed items (`confirmed-done` and `likely-done`) are now removed from PLAN.md outright — `git log` and the changelog are the audit trail for what was finished and when. Phase 4 ("Archive to DONE.md") is deleted; subsequent phases renumber 5→4 (Rebuild PLAN.md), 6→5 (Absorb GOALS.md), 7→6 (Commit). The Phase 6 commit message format now expects the completed slugs spelled out explicitly — `docs: replan — completed [slug-a], [slug-b]; pruned {S} stale, added {P} new` — so `git log --grep=<slug>` becomes the canonical way to find when a slug was retired.
- **Slug uniqueness scan is now PLAN.md-only.** Phase 0 of `do:replan`, plus the concurrent-appender uniqueness checks in `do:better`, `do:better-swift`, `do:depfree`, and `do:goals`, no longer consult `DONE.md`. The canonical spec at [`lib/plan-id-format.md`](../lib/plan-id-format.md) drops both the "DONE.md archival" section and the DONE.md regex from the strict positional pattern. A retired slug *can* theoretically be reused for a brand-new item, but the title-derivation rules make accidental reuse unlikely in practice — reach into `git log --grep=<slug>` if you need to confirm.
- `do:push` slug-grep guidance updated: finished work is now grep-able across the changelog, branches, and PR titles (DONE.md removed from the list).

## Fixed

## Removed

- **DONE.md handling, end-to-end.** Existing `DONE.md` files in user repos are left untouched on disk — slashdo simply no longer reads or writes them. Projects that have one can delete it, keep it as a historical artifact, or maintain it by hand; none of the slashdo commands will touch it going forward.
