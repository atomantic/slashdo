# Release v3.17.1

Released: 2026-06-20

## Review loop
- All reviewer prompts (`copilot`, `codex`, `agy`, `claude`, `ollama`) now tell reviewers to focus on **logic bugs tied to concrete wrong behavior**, not syntax/lint/format/build issues the project's own linter, type-checker, compiler, and test suite already catch — zero findings is the expected outcome. The `ollama` prompt gets the strongest version plus explicit negative examples, since a small local model asked to "review against best practices" otherwise produces a flood of style nits, rename suggestions, and confident "you deleted X, this breaks Y" claims about code it only sees one file's diff of. The canonical code-review checklist leads with the same "review logic, not lint" principle.
- `do:release` now leads the changelog with a human-readable, feature-grouped summary instead of a flat list of every file change, and notes the host release-body size limit (GitHub's 125k-char cap; large bodies can also overflow `ARG_MAX` as an action input).
