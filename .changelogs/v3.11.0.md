# Release v3.11.0

Released: 2026-06-13

## Added

- **Ollama reviewer** (`--review-with=ollama` or `--review-with=ollama[<model>]`): a fully local, offline review pass. Unlike the agentic CLIs, Ollama is non-agentic and review-only — the orchestrator feeds it the per-file diff, parses its findings, then applies, verifies, and pushes the fixes itself. Bare `ollama` auto-selects the most capable installed coding model (preferring coding-specialized families by parameter size); `ollama[<model>]` pins a specific installed model. Partial reviews (per-file errors or diff truncation) report an `incomplete` status that the merge gate treats as inconclusive — never eligible to merge.
- **`/do:config` persistent review defaults**: save the shared review flags (`--review-with`, `--review-iterations`, `--reviewer-applies`, `--review-stop-mode`) as defaults so future commands can omit them. Defaults resolve global (host CLI config) or per-project (`.slashdo.json` via `--project`). Precedence, highest first: explicit flag (or `--review-with none`) > project > global > built-in. A project can opt out of an inherited global reviewer via the `none` tombstone (`--no-reviewer-applies` / `--review-stop-all` cover the other flags).

## Changed

- Reviewer flags are now wired consistently across `/do:pr`, `/do:release`, `/do:rpr`, and `/do:review`, including project-level off-overrides for `reviewer-applies` and stop-mode, and the new defaults-resolution step shared via `lib/review-config-defaults.md`.
