## Fixed

- **`/do:update`: statusline badge now clears immediately after updating.** The installer wrote the new version to `~/.claude/.slashdo-version` but left `~/.claude/cache/slashdo-update-check.json` untouched, so the statusline kept reading the stale cache and showing `⬆ /do:update` for the rest of the session. On the next TUI restart, the async SessionStart `npm view` check raced with statusline render, causing the badge to flash once before the cache was rewritten. The installer now writes a fresh cache file (`update_available: false`, `installed = latest = pkg.version`) right after updating the version file, so the badge clears in the current session and stays gone across restarts.
