# Release v3.21.0

Released: 2026-06-30

## Highlights
- `--review-with` now accepts an **arbitrary GitHub reviewer** via the `@<login>` form — any GitHub user or App/bot (e.g. `--review-with @octocat`, `--review-with claude,@org-review-bot`, `--review-with @some-app[bot]`). slashdo requests that login's review on the PR, waits for it, and fixes whatever it surfaces — the same request → poll → fix → resolve flow the `copilot` reviewer already uses. In `--merge` mode the review must complete for the merge to proceed; a reviewer that never submits leaves the PR open rather than merging on an absent review. slashdo never posts an approval itself.

## Added
- **Arbitrary GitHub reviewers (`@<login>`)** — a new reviewer type accepted everywhere `--review-with` is (`/do:pr`, `/do:release`, `/do:review`, `/do:better`, `/do:better-swift`, `/do:depfree`, `/do:rpr`, and `/do:config` for saved defaults). It generalizes the Copilot loop to any GitHub login (a user, or an App whose login ends in `[bot]`):
  - New shared procedure `lib/github-reviewer-loop.md` requests the review (`gh api .../requested_reviewers`), polls GraphQL for a submitted review from that login, applies fixes, resolves threads, and pushes — bounded by `--review-iterations` and a generous wait suited to slower (human) reviewers.
  - A failed review request is non-fatal: slashdo keeps polling, so review Apps that post on their own (without being explicitly requested) still work.
  - Statuses: `clean`/`capped` (eligible to merge), `timeout`/`not-requestable`/`error`/`guardrail` (inconclusive — never merged).
  - GitHub only; on GitLab the `@<login>` (and `copilot`) passes are skipped with a warning and an `inconclusive` aggregate.
  - Login validation: `^[A-Za-z0-9][A-Za-z0-9-]*(\[bot\])?$`; the login (lowercased) is part of the `--review-with` dedup identity.

## Changed
- `/do:pr` now partitions reviewers into **local** (run before the PR, against the working tree) and **PR-side** (`copilot` + every `@<login>`, run after the PR is opened) — the section formerly titled "Run the Copilot Review" is now "Run the PR-side Reviews". The merge gate is unchanged: it is driven by the aggregate review status, which is reviewer-agnostic.
