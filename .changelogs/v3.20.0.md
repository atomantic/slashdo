# Release v3.20.0

Released: 2026-06-30

## Highlights
- `do:better --issues` now files cleaner tracker issues: finding category and severity move out of the issue title and into native GitHub/GitLab labels, so titles stay readable and you can filter by `severity:*` or category tags.
- The `--review-with=claude` reviewer dispatches the correct sub-agent type directly, so it no longer wastes a turn recovering from a missing `code-reviewer` agent before the review starts.

## Changed
- **Issue labels, not title brackets** ‚Äî `do:better --issues` (and the inlined `plan-issue-mode` shared by `do:better-swift` and `do:next`) stops baking `[category][SEVERITY]` prefixes into issue titles. It now writes a clean title plus a category label and a `severity:critical|high|medium|low` label, creating the labels lazily and idempotently with a severity‚Üícolor map; reused/deduped issues keep their existing labels. (#99)

## Fixed
- **Claude local reviewer names its sub-agent type** ‚Äî the `--review-with=claude` reviewer now specifies the exact sub-agent type to dispatch (`general-purpose`) instead of probing for a non-existent `code-reviewer` agent and burning a turn recovering before the review begins.
