# Release v1.7.0

Released: 2026-03-15

## Added

- `do:better-swift` — SwiftUI-optimized DevSecOps pipeline with multi-platform coverage (iOS, macOS, watchOS, tvOS, visionOS), Swift-specific audit agents, and `swift-review-checklist.md`
- `lib/swift-review-checklist.md` — 4-tier code review checklist for Swift/SwiftUI apps covering runtime safety, async/concurrency, multi-platform parity, accessibility, and SwiftUI patterns
- XcodeGen, TestFlight upload, and iCloud ubiquity container checks added to `do:better-swift` Agent 6 from existing skills
- `install.sh` local mode — uses local files when run from repo directory, falls back to curl from GitHub otherwise
- 23 new checklist items and 10 review deep checks from PR feedback (#88, #94, #95, #516)
- `do:review` — only posts PR comments on reviews of others' PRs
- Entity type-change validation: checklist item + review deep check for invariant revalidation on discriminator changes
- Data migration semantic preservation: checklist item + review deep check for schedule/state/trigger fidelity
- Checklist: caller/callee resolve/reject contract disagreement and async EventEmitter handlers
- Checklist: interactive prompts blocking non-interactive/CI contexts (TTY detection)
- Checklist: security/sanitization functions covering only one input format
- Checklist: unbounded subprocess output buffering, naive command string splitting, lazy-migration concurrency
- Checklist: broadened imports check, coupled-state check, serialization boundary check, prototype pollution, missing-value semantics
- Review process: bulk vs single-item operation parity, config value provenance, self-rescheduling callback resilience, periodic skip behavior, migration idempotency deep checks

## Changed

- Dependabot enabled for GitHub Actions and npm dependencies
- `do:review` now only posts PR comments when the PR was opened by someone other than the current user

## Fixed

- `do:better` — `{FIRST_CATEGORY}` placeholder now explicitly assigned before use in version bump
- `do:better` — Error Recovery iteration cap aligned to 10 (matching Phase 6 guardrail) instead of contradictory 5
- `do:better-swift` — `{BUILD_CMD_per_platform}` undefined placeholder replaced with actual Phase 0c variable references
- `do:better-swift` — `git add -A` replaced with `git add *.plist *.pbxproj` for version bump
