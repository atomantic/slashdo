## Changed

- Trimmed the multi-agent review system from ~320KB / 1,145 lines to ~60KB / 796 lines by rewriting each agent file as mandate + principles + short hot-pattern reminders + concrete past-miss examples. Past misses (fallback shape divergence, encoder corruption of downstream parsers, tests asserting symptoms, LLM prompts promising nonexistent code paths) are consequence-reasoning bugs — adding bullets doesn't catch them; a principled lens does.
- Copilot review loop now treats "exceeds the maximum number of lines (20,000)" as a terminal clean state — no re-request, proceeds directly to merge. Updated `better.md` edge cases and `release.md` merge criteria with the same rule.

## Fixed

- `do:fpr` origin guard now rejects non-GitHub remotes and whitespace-bearing slugs. The previous shape-only check let URLs like `git@gitlab.com:foo/bar.git` pass through (the slug-derivation pipeline is a no-op when the URL lacks `github.com`, so the raw URL was treated as a slug), leading to a confusing `gh repo view` failure.
- Synced `lib/copilot-review-loop.md` with the installed copy so source and shipped instructions match.
